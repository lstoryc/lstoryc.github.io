处理公司上网需要代理导致 Git NPM 无法正常连接外网

#解决公司上网需要设置代理导致 git npm 无法正常使用

####1. git:

######设置git代理，具体代理地址和端口参考贵公司提供的地址
git config –global http.proxy http://10.61.20.200:1081

######查看git全局配置信息（编辑参数为 -e）
git config –global -l

####2. npm

######处理npm install失败，报503错误
npm config set registry=http://registry.npmjs.org

######查看npm全部配置
npm config list -l

######其中有个用户配置信息, 该文件内容为用户配置项

; userconfig C:\Users\xxxx.npmrc
registry = “http://registry.npmjs.org/"
